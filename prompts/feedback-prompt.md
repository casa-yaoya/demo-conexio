# フィードバックプロンプト

## 役割
AIロープレ終了後、会話ログを分析して100点満点で評価する。

## 使用タイミング
各モードのロープレが終了し、会話ログが生成された後

## 入力
- ロープレ設計の全内容（特にポイント）
- ロープレの会話ログ全文
- 使用したモード（subtitle, ai-demo, confirmation, practice）

## プロンプトテンプレート

```
「あなた」が以下のリストの問に対して、正解をヒントをもらわずに答えられているかで採点してください。
正解やヒントを相手からもらっているケースも多くありますが、この場合は減点または０点としてください。
各問に対して採点を行い、全体で100点満点で整数値で採点してください。

[ロープレ設計のポイントから、問と正解基準を挿入]

【会話ログ】
[実際の会話ログを挿入]

【採点基準】
- 各ポイントについて、自力で正解できている場合：満点
- 部分的に正解だが不足がある場合：50〜80%の点数
- ヒントをもらって答えた場合：20〜40%の点数
- 全く答えられなかった、または誤答の場合：0点

【出力形式】
以下の形式でJSON形式で出力してください：

{
  "totalScore": 点数（整数、0〜100）,
  "details": [
    {
      "question": "問の内容",
      "score": 点数,
      "maxScore": 満点,
      "feedback": "評価コメント"
    },
    ...
  ],
  "overallFeedback": "全体的な評価コメント（200文字以内）",
  "strengths": ["良かった点1", "良かった点2"],
  "improvements": ["改善点1", "改善点2"]
}
```

## モード別の評価ポイント

### 台本モード
- 台本通りに話せているか
- 抜けや漏れがないか
- スムーズに進行できたか

### お手本モード / 確認モード
- ポイントを理解しているか
- 自分の言葉で説明できているか
- 正解基準を満たしているか

### 実践モード
- 必須ミッションを達成できたか
- 相手の反応に適切に対応できたか
- 会話技術を効果的に使えたか

## 出力時の注意点
1. 具体的で建設的なフィードバックを提供
2. 良かった点を必ず含める
3. 改善点は実行可能な形で提示
4. 数値だけでなく、言葉での評価も重視
